========================================
MULTI-ROLE ONBOARDING & DYNAMIC UI SYSTEM
========================================

Create a role-based onboarding and UI system where users can select their role during signup/login.

IMPORTANT:
Each role must have:
- Completely different dashboard UI
- Different navigation
- Different permissions
- Different workflows
- Different analytics
- Different features
- Different home screens

The application should dynamically render UI based on user role.

========================================
USER ROLE SELECTION
========================================

During onboarding/signup, show a modern role selection screen.

Roles:
1. Gym Owner / Gym Admin
2. Trainer
3. Gym Member / Fitness User

Use:
- Large modern selection cards
- Icons/illustrations
- Smooth animations
- Role descriptions
- Hover/touch effects

Example:
--------------------------------
Select Your Role

[ Gym Owner ]
Manage your gym, trainers, members, attendance, and analytics.

[ Trainer ]
Manage client workouts, diets, transformations, and progress.

[ Gym Member ]
Track workouts, calories, body progress, and gym activity.
--------------------------------

After role selection:
- Store role in database
- Redirect to role-specific onboarding
- Generate role-specific dashboard

========================================

- Dynamic sidebar menus
